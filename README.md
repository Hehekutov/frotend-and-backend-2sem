# Практики 13-17

В этом репозитории находится мой проект по практическим работам 13-17 по дисциплине "Фронтенд и бэкенд разработка".
Проект выполнил студент ЭФБО-18-24 Кудояров Денис.

Основная работа находится в папке `prac13-17`. В проекте сделано приложение "Заметки", которое постепенно дорабатывалось от обычного веб-приложения до PWA с push-уведомлениями и напоминаниями.

## Что реализовано

- добавление и удаление заметок;
- хранение заметок в `localStorage`;
- офлайн-работа через `Service Worker`;
- `manifest.json` для установки приложения как PWA;
- работа через `HTTPS` на локальном сервере;
- архитектура `App Shell` с отдельной подгрузкой контента;
- push-уведомления;
- отправка событий в реальном времени через `WebSocket`;
- заметки с напоминанием по дате и времени;
- кнопка отложить напоминание на 5 минут.

## Кратко по практикам

### Практика 13. Service Worker

В этой практике было сделано первое офлайн-приложение для заметок. Я реализовал регистрацию `Service Worker`, кэширование основных файлов и работу заметок без интернета. Данные заметок сохраняются в `localStorage`, поэтому после перезагрузки страницы они не пропадают. Также была настроена базовая логика `install`, `activate` и `fetch`.

### Практика 14. Web App Manifest

На этой практике приложение было доработано до формата PWA. Я добавил файл `manifest.json`, мета-теги в `index.html` и набор иконок разных размеров. За счёт этого приложение можно открывать как отдельное установленное веб-приложение. Также были настроены название приложения, цвет темы и стартовая страница.

### Практика 15. HTTPS + App Shell

В этой практике приложение было переведено на архитектуру `App Shell`. Я разделил интерфейс на каркас приложения и динамически подгружаемый контент, который загружается внутри основного контейнера. Также была настроена работа через локальный `HTTPS` с сертификатами `localhost.pem` и `localhost-key.pem`. Это нужно для корректной работы `Service Worker` и остальных PWA-возможностей.

### Практика 16. WebSocket + Push

В этой практике я добавил серверную часть на `Node.js` и `Express`. Для обмена событиями в реальном времени подключил `Socket.IO`, чтобы новые заметки могли сразу появляться у других клиентов. Также была реализована подписка на push-уведомления через `web-push` и `PushManager`. В проекте появились кнопки включения и отключения уведомлений, а подписки сохраняются на сервере.

### Практика 17. Детализация Push

В этой практике я добавил заметки с напоминанием по дате и времени. На клиенте появилась отдельная форма для создания напоминания, а в `localStorage` заметки стали храниться с полями `id` и `reminder`. На сервере реализовано планирование уведомлений через `setTimeout`, чтобы push приходил в нужный момент. В `Service Worker` была добавлена кнопка "Отложить на 5 минут", которая отправляет запрос на сервер для переноса напоминания.

## Какие технологии использовались

- `HTML` - структура страниц;
- `CSS` - оформление интерфейса;
- `JavaScript` - логика на клиенте;
- `Node.js` - запуск серверной части;
- `Express` - сервер и раздача статических файлов;
- `Socket.IO` - обмен событиями между клиентом и сервером в реальном времени;
- `web-push` - отправка push-уведомлений;
- `Service Worker` - кэширование и офлайн-режим;
- `Web App Manifest` - оформление PWA;
- `localStorage` - локальное хранение заметок в браузере.

## Структура проекта

- `prac13-17/index.html` - основная страница приложения;
- `prac13-17/content/` - отдельные части интерфейса;
- `prac13-17/app.js` - клиентская логика;
- `prac13-17/sw.js` - service worker;
- `prac13-17/server.js` - сервер для практик 15-17;
- `prac13-17/manifest.json` - настройки PWA;
- `prac13-17/vapid-keys.json` - VAPID-ключи для push-уведомлений;
- `prac13-17/localhost.pem` и `prac13-17/localhost-key.pem` - сертификаты для локального HTTPS.

## Как запустить

1. Установить зависимости:

```bash
npm install
```

2. Заполнить файл `prac13-17/vapid-keys.json` своими ключами.

3. Запустить проект:

```bash
npm run prac13-17:start
```

4. Открыть адрес, который появится в консоли.

Если сертификаты `localhost.pem` и `localhost-key.pem` лежат в папке `prac13-17`, приложение будет доступно по `HTTPS`.
